# คู่มือโคลนระบบติดตามลูกค้าย้ายฐาน (Template)

เทมเพลตนี้คือระบบเดิม **ตัดข้อมูล + แบรนด์ออกแล้ว** พร้อมนำไปใช้กับเว็บใหม่ (แนวลูกค้าย้ายจากเว็บเก่า → เว็บใหม่)

ไฟล์ในชุด:
- `index_template.html` — ตัวเว็บแอปทั้งหมด (โค้ด+ฟีเจอร์ครบ, ข้อมูลว่าง, แบรนด์เป็น placeholder)
- `worker_template.js` — Cloudflare Worker (ฐานข้อมูลกลาง KV + Telegram + ประตูรหัสผ่าน)

ฟีเจอร์ที่ติดมาครบ: แดชบอร์ด (ล็อกเดือน + ตัวกรองกลุ่มระดับ + ▲▼ เทียบวันก่อน), หน้าติดตามลูกค้า (แบ่งหน้า/ค้นหา/แก้ไข/โบนัส VIP), สรุปผล + รายงานพนักงาน, อัปโหลดยอดรายวัน (Excel), แจ้งเตือน Telegram + ส่งรายงาน (ภาพ+สรุป), URL routing, ประตูรหัสผ่าน (GATE_PASS)

---

## ขั้นตอนโคลน (5 ขั้น)

### 1) เปลี่ยนแบรนด์ (find-replace)
เปิด `index_template.html` แล้วแทนที่ทั้งไฟล์:
- `__SITEA__`  → ชื่อ**เว็บเก่า** (เว็บต้นทางที่ลูกค้าย้ายออก เช่น ADEN168)
- `__SITEB__`  → ชื่อ**เว็บใหม่** (เว็บปลายทางที่ลูกค้าย้ายมา)

จากนั้นเปลี่ยนโลโก้/ไอคอน (ถ้าต้องการ) — ค้นหา `data:image/png;base64,` ในไฟล์ จะเจอ 3 จุด: favicon, โลโก้หน้า login (`id="loginLogo"`), โลโก้แบนเนอร์ (`id="heroRyLogo"`) แทนที่ base64 ด้วยของเว็บใหม่ (ถ้าไม่เปลี่ยน จะเป็นโลโก้เดิม)

แล้ว **เปลี่ยนชื่อไฟล์เป็น `index.html`**

### 2) โหลดข้อมูลลูกค้า — **ทำเองผ่านเว็บได้เลย** (ไม่ต้องฝังในไฟล์อีกต่อไป)
มีการ์ด **"📇 อัปโหลดข้อมูลลูกค้า"** ในหน้าสรุปผลการติดตาม (เฉพาะ Head ขึ้นไป):

**ก. ฐานลูกค้าเว็บเก่า** — กด "อัปโหลดข้อมูลลูกค้า" แล้วเลือกไฟล์ Excel ที่มีคอลัมน์: `เบอร์โทร · ชื่อบัญชี · เลขบัญชี · ยูสเว็บเก่า · ยอดฝาก (4 เดือน)` → ระบบสร้าง `FOLLOW/PIV/DB` ให้อัตโนมัติ · **กดปุ่ม "⬇ ดาวน์โหลดไฟล์ตัวอย่าง" เพื่อดูรูปแบบไฟล์ที่ถูกต้อง** (หรือใช้ไฟล์ `ตัวอย่าง_ข้อมูลลูกค้า.xlsx` ในชุดนี้)

**ข. ยอดกลับมาฝากเว็บใหม่** — ใช้การ์ด "📥 อัปเดตยอดกลับมาฝาก" อัปโหลดไฟล์ยอดฝาก-ถอนรายวัน (จับคู่ด้วยเลขบัญชี) — ระบบ normalize เลขบัญชี (ตัด 0 นำหน้า) จับคู่ 2 ฝั่ง + จัดระดับ (tier) ให้เอง

> ข้อมูลทั้งหมดเก็บบนฐานข้อมูลกลาง (KV) ซิงก์ทุกเครื่อง · ไม่ต้องส่งไฟล์ให้ผมประมวลผลอีก เว้นแต่ไฟล์รูปแบบแปลกที่ต้องแปลงก่อน

### 3) ตั้งโครงสร้างพื้นฐาน (ทำครั้งเดียว)
- **GitHub repo ใหม่ (ต้อง Private — มี PII ลูกค้า):** วาง `index.html` ที่ root และ `worker.js` ที่ `src/worker.js`
- **Cloudflare Worker ใหม่** + **KV namespace ใหม่**
- `wrangler.toml`:
  ```
  name = "ชื่อ-worker-ใหม่"
  main = "src/worker.js"
  compatibility_date = "2026-01-01"
  [assets]
  directory = "./public"
  binding = "ASSETS"
  [[kv_namespaces]]
  binding = "DB"
  id = "ใส่ KV namespace id ใหม่"
  ```
  (วาง `index.html` ไว้ในโฟลเดอร์ `public/`)
- ผูก KV binding ชื่อ **DB** เข้ากับ Worker

### 4) Deploy
อัป `index.html` + `src/worker.js` ขึ้น GitHub → Cloudflare กด **New deployment**

### 5) ความปลอดภัย (สำคัญ)
- ตั้ง secret **`GATE_PASS`** ใน Worker (Settings → Variables and Secrets → Secret) เพื่อกันเว็บด้วยรหัสผ่านชั้นนอก (กัน PII หลุด) — username เริ่มต้น `team` (เปลี่ยนได้ด้วย `GATE_USER`)
- ถ้ามีโดเมนของตัวเอง แนะนำตั้ง **Cloudflare Access** (อีเมล+OTP รายคน) เพิ่ม
